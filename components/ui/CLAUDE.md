# components/ui/ — shadcn/ui primitives + bespoke UI

Read before editing anything here. **Two kinds of files live together** — treat them
differently.

## 1. shadcn/ui-generated primitives
`button`, `card`, `form`, `input`, `table`, `textarea`, `badge`, `dialog`, `select`,
`dropdown-menu`, `popover`, `tabs`, `toast`, … — generated by the shadcn CLI
(`components.json`: `style: "default"`, `baseColor: "slate"`, `rsc: true`, `tsx: true`).

- Prefer **regenerating / updating via the shadcn CLI** over hand-editing, so upgrades
  stay clean. The non-interactive primitives (`button`, `card`, `form`, `input`, `table`,
  `textarea`, `badge`) are server-safe (no `"use client"`); the Radix-backed interactive
  ones carry `"use client"` — keep that split.
- A later phase updates these (and Tailwind), so keep local customizations minimal and
  clearly marked — the regeneration diff needs to stay legible.

## 2. Bespoke / aceternity-style effects
`lamp`, `sparkles`, `wavy-background`, `tracing-beam`, `typewriter-effect`, and the
`mail/` inbox UI are **custom** (Framer Motion / canvas) — **not** shadcn, so the CLI
won't regenerate them. Edit these by hand.

Shared helper for both: `cn()` from `lib/utils.ts`.
